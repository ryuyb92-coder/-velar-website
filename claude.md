# VELAR Website Rules

## Brand Identity

VELAR is a premium mobile detailing brand based in Dallas.

The brand should feel:
- premium
- modern
- trustworthy
- calm
- cinematic
- high-end
- conversion-focused
- luxury without intimidation

This is NOT:
- a tech startup
- a SaaS dashboard
- a cyberpunk luxury brand
- an exotic supercar company

VELAR should feel like:
- a premium local luxury service
- something customers immediately trust
- clean, intentional, and professional
- visually elevated but approachable

Premium means:
- restraint
- spacing
- hierarchy
- readability
- confidence without clutter

  - overall restraint and polish

## Reference Interpretation Rules

All files inside /references are inspiration sources only.

Claude must NEVER:
- copy text directly
- duplicate package names
- recreate layouts 1:1
- imitate competitor branding
- reuse competitor wording
- mirror section structures exactly
- clone visual proportions too closely

References should instead be used to:
- understand premium layout hierarchy
- study spacing rhythm
- study package comparison UX
- understand toggle behavior
- improve card readability
- improve responsive structure
- improve premium presentation

All generated content must be original to VELAR.

VELAR should feel:
- inspired by premium detailing brands
- professionally designed
- intentional
- distinct
- trustworthy
- premium without imitation

Package naming, descriptions, headings, and service explanations must be rewritten uniquely for VELAR.

Vehicle imagery and mockups must:
- use VELAR branding only
- avoid all competitor branding/logos
- maintain VELAR’s stealth-luxury automotive aesthetic
- feel clean, realistic, and premium

The goal is:
premium inspiration → original execution.

NOT:
competitor imitation.

Do NOT copy references directly.

Instead:
- reinterpret them for VELAR
- improve responsiveness
- improve UX clarity
- maintain visual consistency

## UX Philosophy

The website should prioritize:
1. trust
2. clarity
3. readability
4. conversion
5. premium presentation

Every section should answer:
- What is VELAR?
- Why trust VELAR?
- What services are offered?
- How much does it cost?
- How do I book?

Do not sacrifice clarity for aesthetics.

## Pricing Section Rules

Pricing cards should feel:
- premium
- readable
- conversion-focused
- structured
- high-end
- easy to compare quickly

Use:
- strong spacing hierarchy
- clean comparison layouts
- restrained accents
- premium typography
- readable package structures
- subtle emphasis for featured packages

Avoid:
- cluttered comparisons
- oversized badges
- excessive gradients
- noisy backgrounds
- overwhelming feature lists

Package layouts should:
- prioritize readability first
- scan easily on mobile
- maintain visual consistency
- use concise descriptions
- clearly separate included services
- maintain strong CTA visibility

Toggle systems should:
- feel smooth
- remain intuitive
- update pricing clearly
- maintain layout consistency across states

XL toggle pricing should:
- dynamically update package pricing
- maintain identical layout structure
- avoid layout shifting or resizing

Interior & Exterior and Interior Only sections should:
- feel related
- but not duplicated
- maintain unique package positioning
- maintain unique descriptions and hierarchy

## Workflow

Before making major UI changes:

2. Identify affected sections
3. Then implement

Always maintain visual consistency across:
- desktop
- tablet
- mobile

Always reference existing design language before creating new UI patterns.

## V1 Scope Lock

VELAR v1 is intentionally focused and restrained.

Primary goals:
- establish trust
- communicate premium quality
- simplify package selection
- drive bookings
- maintain low cognitive load

The website should prioritize:
1. Hero section
2. Service overview
3. Pricing section
4. Trust/reviews
5. Booking CTA
6. Footer

Avoid:
- unnecessary sections
- feature bloat
- excessive animations
- fake luxury aesthetics
- SaaS-style layouts
- dashboard UI patterns
- overcomplicated booking flows
- paint correction flows
- ceramic coating flows
- popup spam
- urgency gimmicks

Every section must justify its existence.

Whitespace, readability, and clarity are more important than visual complexity.
